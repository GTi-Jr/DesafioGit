O QUE É GIT:
Inicialmente, Git é um controle de versão, mas antes disso, uma apresentação básica: O Git,simplesmente, permite que várias pessoas modifiquem um arquivo/projeto/código/... ao mesmo tempo e permite "voltar no tempo" para corrigir o que fez em algum momento. 
Assim como criamos pastas, colocamos arquivos, modificamo-as e depois salvamos, da mesma forma é com o Git, no entanto, ele oferece as possibilidades já ditas anteriormente. 
Além do mais, conseguimos saber quem modificou o arquivo/..., o que modificou e quando modificou, através de um histórico.
Portanto, com essas informações interpretadas, um controlador de versão é um sistema que nos permite gerenciar as modificações geradas em um arquivo/... ou grupos de arquivos/... com o passar do tempo, permitindo a possibilidade de acessarmos diferentes versões ,antes já criadas, para quando houver essa necessidade.
O QUE É GITHUB:
O GitHub em poucas palavras seria um rede social digital para desevolvedores. Ele serve de hospedagem para códigos para controle de versão (explicado anteriormente) e colaboração entre os desenvolvedores com os códigos compartilhados. E o GitHub nos permite compartilhar e desevolver juntamente com pessoas de qualquer lugar.
Além do mais, pelo fato do GitHub ser uma rede social digital de desenvolvedores, isso a torna essencial para o mercado de trabalho.
